網站載入速度優化方法｜2026年完整技術指南

2026-05-01 ・ HEY!BOSS 銀月數位顧問

網站速度為什麼這麼重要？

根據 Google 研究，頁面載入時間從 1 秒增加到 3 秒，跳出率上升 32%。增加到 5 秒，跳出率暴增 90%。更重要的是，2021 年起 Google 將 Core Web Vitals 納入排名因素，網站速度直接影響你的搜尋排名。

HEY!BOSS 銀月數位顧問建置的網站，平均 Lighthouse 效能分數在 90 分以上。以下分享我們的優化方法論。

一、圖片優化（最大效益）

圖片通常佔網頁總大小的 50~70%，是優化的首要目標。

1. 使用新一代格式：WebP 比 JPEG 小 25~35%，AVIF 更小 50%。使用 <picture> 標籤提供多格式支援。

2. 適當的尺寸：不要上傳 4000px 寬的照片然後用 CSS 縮小到 800px。根據實際顯示尺寸裁切圖片。

3. 懶載入（Lazy Loading）：加入 loading="lazy" 屬性，讓畫面外的圖片延遲載入。首屏圖片不要懶載入。

4. 圖片 CDN：使用 Cloudflare Images、imgix 或 Cloudinary 等服務，自動根據裝置尺寸與網路狀況提供最佳化的圖片。

二、程式碼優化

1. 壓縮 CSS/JS：使用工具（如 terser、cssnano）移除空白與註解，減少檔案大小 30~60%。

2. 移除未使用的 CSS：使用 PurgeCSS 移除未使用的樣式。許多網站有超過 90% 的 CSS 是未使用的。

3. 延遲載入非關鍵 JS：使用 defer 或 async 屬性，讓分析工具、聊天插件等非核心腳本不阻塞渲染。

4. 減少第三方腳本：每個第三方腳本（廣告、追蹤碼、社群插件）都有成本。定期審視是否每個都真的需要。

5. 程式碼分割（Code Splitting）：只載入當前頁面需要的 JS，其他頁面的程式碼延後載入。

三、伺服器與基礎設施

1. CDN（Content Delivery Network）：使用 Cloudflare、AWS CloudFront 等 CDN，將靜態資源分發到全球節點。台灣使用者連到新加坡或東京節點，比連到美國快 10 倍。

2. HTTP/2 或 HTTP/3：確保伺服器支援 HTTP/2 以上協定，允許多路復用，大幅減少連線建立時間。

3. Gzip/Brotli 壓縮：啟用伺服器端壓縮，HTML/CSS/JS 檔案可再縮小 60~80%。Brotli 比 Gzip 壓縮率更好。

4. 瀏覽器快取設定：為靜態資源設定適當的 Cache-Control 標頭，讓回訪使用者不需重新下載。

5. 伺服器回應時間（TTFB）：目標在 200ms 以內。如果超過 600ms，考慮升級主機或優化後端程式碼。

四、Core Web Vitals 三大指標

LCP（Largest Contentful Paint）：最大內容繪製時間，目標 2.5 秒內。通常是首屏大圖或標題文字。優化方法：預載入關鍵資源、優化圖片、減少伺服器回應時間。

INP（Interaction to Next Paint）：互動回應速度，目標 200ms 內。優化方法：減少主執行緒阻塞時間、分割長任務、最佳化事件處理器。

CLS（Cumulative Layout Shift）：累積版面位移，目標 0.1 以下。優化方法：為圖片設定固定尺寸、避免動態插入內容、字型預載入。

五、測試工具與監控

測試工具：

  • Google PageSpeed Insights：綜合效能報告 + Core Web Vitals
  • GTmetrix：詳細的載入瀑布圖分析
  • WebPageTest：多地點、多裝置測試
  • Chrome DevTools Lighthouse：本地開發即時測試

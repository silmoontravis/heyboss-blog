網站改版為什麼會「掉流量」？

網站改版是企業成長過程中的必經之路，但執行不當可能讓 Google 排名與自然流量在一夜之間暴跌 50% 以上。最常見的原因是：URL 結構變更但未做 301 轉址、舊內容被刪除、meta 標籤遺失、網站速度變慢等。

HEY!BOSS 銀月數位顧問在協助客戶改版時，會嚴格遵循以下檢查清單，確保改版後的流量平穩過渡。

改版前：準備工作

1. 完整備份舊網站：包含資料庫、所有檔案、圖片素材。確保出問題時可以快速回滾。

2. 匯出 SEO 基線數據：從 Google Search Console 匯出所有頁面的排名、點擊數、曝光數。從 Google Analytics 匯出各頁面的流量數據。這是改版後比較成效的基準。

3. 建立 URL 對照表：將舊網站所有 URL 與對應的新 URL 一一列出。這是 301 轉址的依據，也是最容易被忽略的步驟。

4. 保留高流量頁面的內容：即使設計要全面更新，SEO 表現好的頁面內容（文字、標題、meta）不要隨意大改。

5. 通知 Google：如果改版期間需要暫時下線，使用 503 狀態碼（非 404），讓 Google 知道這是暫時性的。

改版中：技術執行要點

1. 301 轉址設定：每一個舊 URL 都必須設定 301 永久轉址到對應的新 URL。不要用 302 暫時轉址。批量設定可在 .htaccess（Apache）或 nginx.conf 中處理。

2. 保留頁面的 SEO 元素：

  • Title tag：保持核心關鍵字不變
  • Meta description：可微調但保留關鍵字
  • H1 標題：主要關鍵字不要改動
  • 圖片 ALT 文字：不要清空
  • 內部連結結構：確保重要頁面仍有足夠的內部連結指向

3. 確保行動裝置相容：新網站必須通過 Google 的行動裝置友善測試。

4. 結構化資料移植：將舊站的 JSON-LD Schema 標記遷移到新站，確保豐富搜尋結果不會消失。

5. XML Sitemap 更新：產生新的 sitemap.xml，確認所有新 URL 都已包含，且移除已不存在的舊 URL。

改版後：上線檢查清單

上線第一天：

  • 逐一檢查 301 轉址是否正確運作
  • 使用 Screaming Frog 或類似工具爬取全站，確認沒有 404 錯誤
  • 提交新的 sitemap.xml 到 Google Search Console
  • 確認 Google Analytics 追蹤碼正常運作
  • 確認 robots.txt 沒有擋住重要頁面

上線第一週：

  • 每日監控 Google Search Console 的索引狀態與錯誤報告
  • 觀察核心關鍵字的排名變化
  • 確認 Core Web Vitals 分數正常

上線第一個月：

  • 比較改版前後的流量數據
  • 處理任何新發現的 404 或爬取錯誤
  • 如有排名下降，分析原因並進行調整

常見改版地雷

地雷一：換網域但沒做 301 轉址 → 所有 SEO 權重歸零。

地雷二：刪除舊文章頁面 → 已獲得的反向連結全部失效。

地雷三：新網站用 JavaScript 渲染但沒做 SSR → Google 爬蟲抓不到內容。

地雷四：改版期間用 noindex 標籤，上線後忘記移除 → 整站從搜尋結果消失。

地雷五：HTTP 與 HTTPS 混用 → 產生重複內容問題。

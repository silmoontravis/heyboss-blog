為什麼網站一定要有 SSL 憑證？

SSL（Secure Sockets Layer）憑證是網站安全的基礎。安裝 SSL 後，網站網址會從 http:// 變成 https://，瀏覽器也會顯示鎖頭圖示，代表資料傳輸經過加密。

沒有 SSL 的後果：

  • Chrome 會顯示「不安全」警告，嚇跑訪客
  • Google 明確將 HTTPS 列為排名訊號，沒有 SSL 排名會受影響
  • 無法使用 HTTP/2 協定（影響網站速度）
  • 表單資料（帳密、信用卡）以明文傳輸，有被竊取風險

SSL 憑證的種類比較

DV（Domain Validation）：僅驗證網域所有權。最快幾分鐘核發。適合個人網站、部落格。免費方案如 Let's Encrypt 即屬此類。

OV（Organization Validation）：驗證組織身份。需提供公司證明文件，1~3 天核發。適合企業官網。年費約 2,000~8,000 元。

EV（Extended Validation）：最高等級驗證。需嚴格審查公司資訊。適合金融機構、電商平台。年費約 8,000~30,000 元。

萬用字元（Wildcard）：可保護主網域與所有子網域（如 *.example.com）。適合有多個子網域的企業。

免費方案：Let's Encrypt 安裝教學

Let's Encrypt 是全球最受歡迎的免費 SSL 憑證機構，每 90 天自動更新。

使用 Certbot 安裝（Linux 環境）：

  1. 安裝 Certbot：sudo apt install certbot python3-certbot-nginx
  2. 申請憑證：sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com
  3. Certbot 會自動修改 Nginx 設定並啟用 HTTPS
  4. 設定自動更新：sudo certbot renew --dry-run 確認排程正常

cPanel 虛擬主機：多數主機商（如 Bluehost、SiteGround）已內建 Let's Encrypt 一鍵安裝功能，在 SSL/TLS 管理介面即可啟用。

付費憑證安裝步驟

  1. 產生 CSR（Certificate Signing Request）：在伺服器上使用 OpenSSL 產生私鑰與 CSR 檔案
  2. 向憑證機構購買：將 CSR 提交給 DigiCert、Comodo、GlobalSign 等機構
  3. 完成驗證：依照 DV/OV/EV 不同等級完成對應驗證流程
  4. 下載憑證檔案：取得主憑證（.crt）與中繼憑證（CA Bundle）
  5. 安裝至伺服器：將憑證與私鑰設定到 Nginx/Apache 的 SSL 設定區段
  6. 設定 HTTP 強制轉向 HTTPS：確保所有 HTTP 請求都被 301 導向 HTTPS

安裝後的檢查清單

  • 使用 SSL Labs（ssllabs.com/ssltest）測試，目標 A 或 A+ 評等
  • 確認所有頁面都走 HTTPS，沒有混合內容（Mixed Content）警告
  • 更新 sitemap.xml 中的 URL 為 https://
  • 在 Google Search Console 中新增 HTTPS 版本的網站
  • 更新所有內部連結、圖片路徑為 HTTPS
  • 確認 canonical 標籤使用 HTTPS URL
  • 設定 HSTS（HTTP Strict Transport Security）強化安全性

常見問題排除

Q：安裝後瀏覽器仍顯示不安全？通常是頁面中有 HTTP 資源（圖片、CSS、JS），需逐一修正為 HTTPS。

Q：憑證過期怎麼辦？Let's Encrypt 設定自動更新即可。付費憑證需在到期前 30 天手動續約。

Q：多個網域需要多張憑證嗎？可使用 SAN（Subject Alternative Name）或 Wildcard 憑證統一管理。

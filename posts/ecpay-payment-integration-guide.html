綠界金流串接完整指南：讓你的網站開始收款

Travis Chen 2026-04-28 9 分鐘閱讀

你的網站做好了、產品上架了、流量也來了，然後客戶問：「我要怎麼付錢？」

如果你的答案是「請匯款到這個帳號，然後截圖給我」，那你正在把客戶推走。現代消費者期望的是點兩下就能完成付款的體驗。任何多餘的步驟，都會讓轉換率下降。

在台灣，綠界科技（ECPay）是中小企業金流串接的首選。門檻低、文件齊全、支援的付款方式最多。這篇文章會帶你從零了解整個串接流程。

一、為什麼線上收款很重要

根據經濟部統計，台灣電商市場規模在 2025 年已超過 5,000 億台幣，而且每年以 10-15% 的速度成長。就算你不是電商，線上收款也越來越重要：

  • 課程報名 — 補習班、工作坊、線上課程的報名費收取
  • 活動票券 — 講座、展覽、社團活動的線上售票
  • 預約訂金 — 餐廳、民宿、沙龍的預約訂金收取
  • 會員訂閱 — SaaS 服務、內容平台的月費/年費收取
  • 捐款 — 非營利組織、社團法人的線上捐款

只要你有「收錢」的需求，金流串接就是必要的基礎建設。而綠界 ECPay 讓這件事變得比你想像的簡單。

二、綠界 ECPay 支援的付款方式

綠界是台灣支援付款方式最完整的金流平台之一：

  • 信用卡 — VISA、MasterCard、JCB，支援一次付清和分期付款（3/6/12/18/24 期）。手續費約 2.75%。
  • 網路 ATM — 消費者在網銀頁面完成轉帳。手續費約 1%（最低 10 元）。
  • ATM 櫃員機 — 系統產生虛擬帳號，消費者到 ATM 或網銀轉帳。手續費約 1%（最低 10 元）。
  • 超商代碼繳費 — 系統產生繳費代碼，消費者到 7-11、全家等超商繳費。手續費約 2%（最低 25 元）。
  • 超商條碼繳費 — 產生條碼讓消費者到超商掃碼繳費。
  • LINE Pay — 串接 LINE Pay 讓消費者用 LINE 錢包付款。
  • Apple Pay / Google Pay — 行動裝置快速結帳。

對消費者來說，付款方式越多，放棄結帳的機率越低。有些客戶習慣刷卡、有些習慣 ATM 轉帳、年輕人喜歡 LINE Pay。綠界一次整合，你全部都有。

三、串接流程概述：從申請到上線

完整的金流串接流程分為四個階段：

階段一：申請綠界帳號

  • 到綠界官網申請商店帳號（個人或公司都可以）
  • 準備資料：營業登記證（公司）或身分證（個人）、銀行帳戶
  • 審核時間約 3-5 個工作天
  • 審核通過後取得 MerchantID（特店編號）和 HashKey / HashIV（加密金鑰）

階段二：API 串接開發

綠界提供 RESTful API，主要串接三支 API：

  • 建立訂單 API — 你的網站產生訂單資訊（金額、商品名稱、訂單編號），送到綠界。綠界回傳付款頁面的 HTML，消費者在綠界的頁面上完成付款。
  • 付款結果通知 API（ReturnURL） — 消費者付款完成後，綠界會打你設定的 ReturnURL，通知你付款結果。你的後端在這裡更新訂單狀態。
  • 查詢訂單 API — 主動查詢某筆訂單的付款狀態。用於對帳和異常處理。

階段三：測試環境驗證

綠界提供完整的測試環境（Sandbox），讓你在不花真錢的情況下模擬完整的付款流程：

  • 使用測試用的 MerchantID 和金鑰
  • 模擬信用卡付款（測試卡號：4311-9522-2222-2222）
  • 驗證 ReturnURL 是否正確接收付款通知
  • 確認訂單狀態更新邏輯是否正確

階段四：正式上線

  • 將 MerchantID 和金鑰切換為正式環境的值
  • API endpoint 從 payment-stage.ecpay.com.tw 切換到 payment.ecpay.com.tw
  • 做一筆小額真實交易驗證
  • 設定好對帳機制和異常告警

四、安全性考量：不能省的細節

金流串接牽涉到「錢」，安全性是最重要的考量。以下幾個環節絕對不能馬虎：

  • HTTPS / SSL 憑證 — 你的網站必須使用 HTTPS。綠界的 ReturnURL 也只接受 HTTPS。這是基本中的基本，沒有 SSL 的網站不應該做任何金流操作。
  • CheckMacValue 驗證 — 每一筆交易都有一個 CheckMacValue，它是用你的 HashKey 和 HashIV 加密計算出來的。你的後端在收到 ReturnURL 通知時，必須重新計算並驗證這個值，確保通知確實來自綠界，而不是有人偽造的。
  • 3D 驗證（3D Secure） — 信用卡交易建議開啟 3D 驗證。消費者在付款時需要輸入銀行發送的驗證碼，防止盜刷。雖然會增加一個步驟，但能大幅降低爭議退款的風險。
  • 金鑰保管 — HashKey 和 HashIV 是你的金流密碼，絕對不能放在前端程式碼裡。必須存在後端的環境變數中（.env），不能 commit 到 Git repository。
  • 訂單編號防重複 — 每筆訂單必須有唯一的 MerchantTradeNo。如果重複，綠界會拒絕交易。建議用時間戳記 + 隨機碼組合。

五、實際應用：RPG 教育平台的金流整合

HEY!BOSS 實際經手的案例中，有一個遊戲化教育平台需要收取課程費用。這個平台把學習內容包裝成 RPG 冒險任務，學員完成任務獲得經驗值和獎勵。

金流需求包括：

  • 課程購買 — 學員購買不同等級的課程包（基礎/進階/大師），價格從 990 到 9,900 元
  • 虛擬寶物購買 — 學員可以用真實貨幣購買遊戲內的道具和裝備
  • 訂閱制 — 月費 299 元的 VIP 會員，享有額外學習資源

我們選擇綠界的原因：

  • 學員年齡層廣（12-45 歲），需要多種付款方式。年輕學員沒有信用卡，用超商代碼繳費；上班族用信用卡分期
  • 綠界支援定期定額扣款，適合訂閱制的自動續約
  • API 文件完整，開發團隊在 5 個工作天內完成串接

上線後，超商代碼繳費佔了 35% 的交易量，驗證了我們提供多元付款方式的決策是正確的。如果當初只支援信用卡，這 35% 的營收可能就流失了。

結語：金流串接不難，但細節很重要

金流串接的技術難度其實不高，綠界的文件和 SDK 已經做得很完善。真正困難的是細節：安全性設定要做到位、異常處理要考慮周全、對帳機制要能追蹤每一分錢。

如果你不確定自己能不能處理這些細節，找一個有經驗的開發團隊幫你串接，是最省時省力的選擇。畢竟金流出錯的代價，可能比開發費用高出好幾倍。

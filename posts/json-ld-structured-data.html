JSON-LD 結構化資料入門：讓 Google 更懂你的網站

Travis Chen 2026-05-10 8 分鐘閱讀

你有沒有在 Google 搜尋時看過一些特別的搜尋結果？有的會顯示星星評分、有的會直接展開常見問題、有的會秀出食譜的烹飪時間。這些都不是 Google 自己變出來的 — 而是網站透過結構化資料（Structured Data）主動告訴 Google 的。

而 JSON-LD 就是 Google 官方推薦的結構化資料格式。

一、什麼是 JSON-LD？

JSON-LD 全名是 JavaScript Object Notation for Linked Data。簡單來說，它是一段放在 HTML <script> 標籤裡的 JSON 格式資料，用來告訴搜尋引擎：「這個頁面的內容是什麼、誰寫的、什麼時候發布的、屬於什麼類型。」

跟其他結構化資料格式（如 Microdata 和 RDFa）相比，JSON-LD 最大的優勢是：完全不需要修改 HTML 內容。你只要在 <head> 裡加一段 <script type="application/ld+json">，搜尋引擎就能理解你的頁面。

這也是為什麼 Google 官方明確表示：「我們推薦使用 JSON-LD」。

二、四種最實用的 Schema 類型

Schema.org 定義了數百種結構化資料類型，但對大多數企業網站來說，以下四種最實用：

1. Organization — 告訴 Google 你是誰

這是最基本的結構化資料。放在首頁，告訴 Google 你的公司名稱、Logo、聯繫方式、社群媒體連結。

當有人搜尋你的公司名稱時，Google 知識面板（Knowledge Panel）就可能顯示這些資訊。

2. LocalBusiness — 在地商家必備

如果你有實體店面或服務據點，LocalBusiness 結構化資料可以告訴 Google 你的地址、營業時間、電話號碼、服務範圍。

這對「附近的餐廳」、「台北網頁設計」這類在地搜尋特別有幫助。Google 地圖和本地搜尋結果都會參考 LocalBusiness 結構化資料。

3. Article — 讓文章內容被正確理解

部落格、新聞、教學文章都適用。Article 結構化資料包含標題、作者、發布日期、文章分類、字數等資訊。

在 HEY!BOSS 的部落格中，每篇文章都包含完整的 Article JSON-LD，讓 Google 能準確理解文章的主題和作者資訊。你現在正在閱讀的這篇文章就有。

4. FAQPage — 搶佔搜尋結果版面

FAQPage 是最能立即提升搜尋結果曝光度的結構化資料。當 Google 認可你的 FAQPage 結構化資料時，搜尋結果會直接展開問答內容，佔據更大的版面。

HEY!BOSS 的每篇文章都包含 3 組常見問題的 FAQPage 結構化資料，這讓我們的文章在搜尋結果中有更高的點擊率。

三、JSON-LD 實作範例

以一個 Article 結構化資料為例：

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "你的文章標題",
  "description": "文章摘要描述",
  "datePublished": "2026-05-10",
  "author": {
    "@type": "Person",
    "name": "作者名稱"
  },
  "publisher": {
    "@type": "Organization",
    "name": "公司名稱"
  }
}
</script>

就是這麼簡單。把這段 JSON 放進 HTML 的 <head> 裡就完成了。

四、HEY!BOSS 如何在 100 個網站實作 JSON-LD

手動為一個網站加 JSON-LD 不難，但如果要為 100 個品牌網站都加上正確的結構化資料呢？

在 HEY!BOSS 銀月數位顧問，我們用 AI 自動化解決了這個問題：

  2. 自動填充 — AI 根據品牌資料自動填入公司名稱、Logo URL、聯繫資訊

五、常見錯誤與注意事項

雖然 JSON-LD 的語法很簡單，但實作時有幾個常見錯誤要避免：

  • JSON 格式錯誤 — 少一個逗號或多一個括號就會讓整段結構化資料失效。建議用 Google 的 Rich Results Test 工具驗證
  • 內容不一致 — JSON-LD 裡寫的標題跟頁面實際標題不一樣，Google 可能會忽略你的結構化資料
  • 虛構資訊 — 在 JSON-LD 裡寫假的星星評分或不存在的 FAQ，Google 會處罰你的網站
  • 遺漏必要欄位 — 每種 Schema 類型都有必填欄位，少了就不會被認可

六、從零開始的行動清單

如果你的網站還沒有任何結構化資料，建議按照這個順序開始：

  1. 首頁加 Organization — 公司名稱、Logo、官網 URL、社群連結
  2. 如果有實體據點，加 LocalBusiness — 地址、電話、營業時間
  3. 部落格文章加 Article + FAQPage — 每篇文章附上 3 個常見問題
  4. 用 Rich Results Test 驗證 — 確認所有結構化資料都被正確解析
  5. 在 Search Console 查看成效 — 觀察「增強」報告中的結構化資料狀態
「結構化資料不會直接提升排名，但它讓你的搜尋結果更吸引人、佔據更大版面、獲得更高點擊率。在 SEO 的世界裡，點擊率就是一切。」

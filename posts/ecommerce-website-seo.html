電商網站 SEO 怎麼做？結構化資料完整教學

Travis Chen 2026-04-26 8 分鐘閱讀

電商網站的 SEO 跟一般品牌網站不一樣。除了關鍵字優化和內容經營，電商 SEO 還有一個被大量賣家忽略但極度重要的武器：結構化資料（Structured Data）。

結構化資料能讓你的商品在 Google 搜尋結果中顯示價格、評價星級、庫存狀態等豐富資訊，讓搜尋結果比競爭對手更吸睛、更容易被點擊。這篇文章，HEY!BOSS 銀月數位顧問完整教學電商網站的結構化資料如何實作。

一、什麼是結構化資料？

結構化資料是一種標準化的資料格式，用來告訴搜尋引擎「這個頁面的內容是什麼」。它不是給人看的（訪客在網頁上看不到），而是給 Google 機器人看的。

Google 推薦使用 JSON-LD 格式來撰寫結構化資料，它被放在 HTML 的 <script> 標籤中。好處是：

  • 不影響頁面外觀 — 純資料，不會改變網頁的 HTML 結構
  • 容易維護 — 集中在一段 JSON 中，修改方便
  • Google 首選 — Google 官方明確推薦 JSON-LD 格式

二、Product Schema：電商的核心結構化資料

對電商網站來說，最重要的結構化資料是 Product schema。它告訴 Google：這個頁面上有一個商品，它的名稱、價格、品牌、庫存狀態、評價分別是什麼。

一個完整的 Product schema 應該包含：

  • name — 商品名稱
  • description — 商品描述
  • image — 商品圖片 URL
  • brand — 品牌名稱
  • sku — 商品編號
  • offers — 價格、幣別、庫存狀態、賣家資訊
  • aggregateRating — 評價星級和評論數量

當 Google 讀懂了你的 Product schema，你的搜尋結果就有機會顯示豐富摘要（Rich Snippets）— 在搜尋結果中直接顯示價格和星級評價，點擊率平均能提升 20-30%。

三、BreadcrumbList Schema：麵包屑導航

麵包屑導航（Breadcrumb）是指網頁上方的路徑導航，例如：首頁 > 3C 配件 > 手機殼 > iPhone 16 手機殼。

在 HTML 中加入 BreadcrumbList schema，Google 搜尋結果會把你的 URL 顯示成有層級的路徑，而不是一長串的網址。好處：

  • 更容易理解 — 用戶一眼就知道這個商品在什麼分類下
  • 更容易點擊 — 用戶可能直接點擊上層分類
  • 幫助 Google 理解網站結構 — Google 更能理解你的頁面層級關係

四、FAQ Schema：常見問題結構化資料

如果你的商品頁面有 Q&A 或 FAQ 區塊，加入 FAQPage schema 能讓這些問答直接顯示在搜尋結果中，大幅增加搜尋結果的佔版面積。

常見的商品 FAQ 包括：

  • 「這個商品有保固嗎？」
  • 「可以退換貨嗎？」
  • 「運費怎麼計算？」
  • 「有什麼付款方式？」

每個 FAQ 問答都是一次出現在搜尋結果中的機會。HEY!BOSS 在每個客戶的網站中都會加入 FAQPage schema，包含行業相關的常見問題。

五、Sitemap：讓 Google 不遺漏任何頁面

電商網站的頁面數量通常很多 — 幾百到幾萬個商品頁面都有可能。如果沒有 sitemap，Google 可能只索引了你的一部分頁面，其餘的商品根本不會出現在搜尋結果中。

一個好的 sitemap 應該：

  1. 包含所有商品頁面 — 不能遺漏
  2. 標示最後更新日期 — 讓 Google 知道哪些頁面最近更新過
  3. 設定更新頻率 — 商品頁面通常是 weekly，首頁是 daily
  4. 自動更新 — 新增或刪除商品時，sitemap 要自動更新

六、Google Search Console 串接

光做好結構化資料還不夠，你還需要透過 Google Search Console 確認一切運作正常：

  • 提交 sitemap — 在 Search Console 中提交你的 sitemap URL
  • 檢查索引狀態 — 確認所有頁面都被正確索引
  • 結構化資料報告 — 查看是否有結構化資料的錯誤或警告
  • 搜尋效能 — 監控豐富摘要的曝光次數和點擊率

HEY!BOSS 在網站上線時會自動串接 Google Search Console，並設定每日監控。如果結構化資料出現錯誤，系統會透過 Telegram 即時通知。

七、完整的電商 SEO 檢查清單

最後，整理一份完整的電商 SEO 檢查清單：

  1. 每個商品頁面都有 Product schema（含 offers 和 aggregateRating）
  2. 網站有 BreadcrumbList schema
  3. 重要頁面有 FAQPage schema
  4. Sitemap 已建立並自動更新
  5. Google Search Console 已串接並提交 sitemap
  6. 所有頁面都有 HTTPS
  7. 每個頁面都有獨立的 title 和 meta description
  8. 圖片都有 alt 屬性
  9. 頁面載入速度在 3 秒以內
  10. Mobile-First 設計完成
「結構化資料不是技術花招，是你對搜尋引擎說的語言。說得越清楚，Google 就越能幫你把商品展示在對的人面前。」
